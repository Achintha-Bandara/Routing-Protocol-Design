━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. ANIMATIONS (Truly Unique Per Protocol)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

OSPF - HELLO Packet Wave Flooding:
├─ Pattern: BFS wave-based spreading
├─ Speed: FAST (0.6s offset per wave)
├─ Behavior: All neighbors at same depth get packets simultaneously
├─ Characteristics:
│  ├─ Wave 1: Source to adjacent routers (0s)
│  ├─ Wave 2: Adjacent to their neighbors (0.6s)
│  ├─ Wave 3: And so on (1.2s, 1.8s...)
│  └─ Return packets staggered within each wave
└─ Real Protocol: Simulates OSPF HELLO mechanism

BGP - UPDATE Message Propagation:
├─ Pattern: Path-selective with policy delays
├─ Speed: SLOW (0.5s base + individual stagger)
├─ Behavior: Each router selects which neighbors get updates
├─ Characteristics:
│  ├─ Policy-based selection (simulated)
│  ├─ Slower propagation than OSPF
│  ├─ Staggered within each hop level
│  └─ Return paths offset differently
└─ Real Protocol: Simulates BGP UPDATE advertisement

RIP - Periodic RESPONSE Updates:
├─ Pattern: Sequential hop-by-hop distribution
├─ Speed: VERY SLOW (0.7s offset between hops)
├─ Behavior: Information spreads one hop at a time
├─ Characteristics:
│  ├─ Hop 0→1: First batch (0s)
│  ├─ Hop 1→2: After delay (0.7s)
│  ├─ Hop 2→3: After longer delay (1.4s)
│  ├─ Each hop takes ~0.7s to propagate
│  └─ Very slow, simulating 30-second RIP timers
└─ Real Protocol: Simulates RIP periodic updates

IS-IS - LSP Flooding (Two Phases):
├─ Phase 1: L2 (Backbone) Flooding
│  ├─ Pattern: Wave-based like OSPF
│  ├─ Speed: MEDIUM (0.35s offset)
│  └─ Behavior: All L2 routers flood simultaneously
├─ Phase 2: L1 (Area) Flooding
│  ├─ Pattern: Sequential from L2 routers
│  ├─ Speed: SLOW (0.3s offset within areas)
│  └─ Behavior: Areas receive info after backbone
└─ Real Protocol: Simulates IS-IS hierarchical LSP flooding

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
