ANIMATION SPECIFICATIONS
========================

Animation Timing Comparison:

OSPF (HELLO Packets - Wave-based):
├─ Wave 0 (0.0s):     Source router
├─ Wave 1 (0.6s):     Direct neighbors (outbound)
├─ Wave 1 (1.0s):     Direct neighbors (return)
├─ Wave 2 (1.2s):     Second-degree routers
├─ Completion (1.8s): All packets delivered, links orange
└─ Total Duration:    ~1.8 seconds

BGP (UPDATE Messages - Policy-based):
├─ Wave 1 (0.0s):     Source router, policy check begins
├─ Wave 2 (0.5s):     First neighbors (staggered 0s-0.15s)
├─ Wave 2 (1.4s):     Return packets
├─ Wave 3 (1.0s):     Second-degree routers
├─ Continuation:       Slower due to policy delays
├─ Completion (2.8s): All packets delivered, links orange
└─ Total Duration:    ~2.8 seconds

RIP (RESPONSE - Sequential):
├─ Hop 0→1 (0.0s):    Source to neighbors
├─ Hop 1→2 (0.7s):    Wait (simulating 30s timer)
├─ Hop 1→2 (1.0s):    Return from hop 1
├─ Hop 2→3 (1.4s):    Wait again
├─ Hop 3→4 (2.1s):    Continues...
├─ Completion (3.0s): All packets delivered, links orange
└─ Total Duration:    ~3.0 seconds

IS-IS (LSP Flooding - Hierarchical):
├─ L2-Phase (0.0s):   Backbone routers flood LSPs
├─ L2-Wave1 (0.35s):  L2 backbone Wave 1
├─ L2-Wave2 (0.70s):  L2 backbone Wave 2
├─ L1-Phase (1.5s):   Area flooding begins
├─ L1-Wave1 (1.5s):   L1 routers receive
├─ L1-Wave2 (1.8s):   L1 continues flooding
├─ Completion (2.2s): All packets delivered, links orange
└─ Total Duration:    ~2.2 seconds
