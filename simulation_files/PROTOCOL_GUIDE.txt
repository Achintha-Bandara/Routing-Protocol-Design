"""
PROTOCOL SIMULATOR USER GUIDE
==============================

This simulator demonstrates 4 different routing protocols with unique animations
and routing table formats for each protocol type.

HOW TO SWITCH PROTOCOLS
=======================
1. Open main.py in the editor
2. Find line 18: PROTOCOL_NAME = 'ospf'
3. Change 'ospf' to one of: 'bgp', 'isis', 'rip'
4. Save and run: python main.py

PROTOCOL SPECIFICATIONS
======================

1. OSPF (Open Shortest Path First)
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   - Animation Type: HELLO Packets (Wave-based flooding)
   - Speed: FAST (0.6s offset between waves)
   - Metric: Cost (link weights)
   - Routing Table Format:
     * Destination | Cost (Metric)
   - Key Features:
     * Rapid wave-based adjacency discovery
     * Simultaneous HELLO flooding to all neighbors
     * Link-state database building
     * SPF (Shortest Path First) calculation
   - Header Color: RED (#e74c3c)

2. BGP (Border Gateway Protocol)
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   - Animation Type: UPDATE Messages (Path-selective propagation)
   - Speed: SLOW (0.5s offset with policy-based delays)
   - Metric: AS Path (autonomous system path length)
   - Routing Table Format:
     * Network | AS Path | Next Hop
   - Key Features:
     * Policy-driven selective route propagation
     * Inter-domain routing
     * Path-based decisions
     * Slower convergence due to policy checks
   - Header Color: BLUE (#3498db)

3. RIP (Routing Information Protocol)
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   - Animation Type: RESPONSE/UPDATE (Sequential hop-by-hop)
   - Speed: VERY SLOW (0.7s offset between hops)
   - Metric: Hop Count (max 15 hops)
   - Routing Table Format:
     * Destination | Metric (Hops) | Status
   - Key Features:
     * Distance-vector algorithm
     * Periodic updates (simulating 30-second timers)
     * Hop-by-hop propagation
     * Unreachable after 15 hops
     * Slowest convergence time
   - Header Color: ORANGE (#f39c12)

4. IS-IS (Intermediate System to Intermediate System)
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   - Animation Type: LSP Flooding (Level-based)
   - Speed: MEDIUM (0.35s L2 backbone, 0.3s L1 areas)
   - Metric: Cost (link metrics)
   - Routing Table Format:
     * Destination | Cost | Level (L1/L2)
   - Key Features:
     * Two-level hierarchy (L2 backbone + L1 areas)
     * LSP (Link State PDU) flooding
     * L2 floods first (backbone), then L1 (areas)
     * Similar to OSPF but with area levels
   - Header Color: PURPLE (#9b59b6)

ANIMATION VISUALIZATION
=======================

During Animation:
  • RED DOTS = Protocol-specific messages traveling across links
  • GRAY LINKS = Inactive network paths
  
After Animation:
  • ORANGE LINKS = Paths used by the protocol
  • Message types shown in title based on protocol

ROUTING TABLES
==============

Each protocol shows its native routing table format:
  • OSPF: Uses link costs from network
  • BGP: Simulates AS paths (AS100, AS200, etc.)
  • RIP: Shows hop counts (0-15 range)
  • IS-IS: Shows hierarchical levels (L1/L2)

HOW TO USE THE SIMULATOR
========================

1. Start the application:
   python main.py

2. Select a source router from the dropdown

3. Click "Run [PROTOCOL]" button

4. Watch the animation showing how the protocol propagates:
   - OSPF: Quick wave-based spreading
   - BGP: Slow selective path propagation
   - RIP: Very slow hop-by-hop spreading
   - IS-IS: Two-phase level-based spreading

5. After animation completes:
   - All links turn ORANGE
   - View protocol-specific routing table
   - Click on routers to see their routing tables

6. Note the differences in:
   - Animation speed (RIP is slowest, OSPF/IS-IS fastest)
   - Message propagation patterns
   - Routing table formats
   - Convergence descriptions

FEATURES
========

✓ Real-time network topology visualization
✓ Protocol-specific animations
✓ Native routing table formats
✓ Live JSON configuration reloading
✓ Convergence time measurement
✓ Per-router routing table display
✓ Protocol-specific convergence logs

CONFIGURATION
=============

Edit network_config.json to change:
  • Network name and description
  • Router count and names
  • Link costs/weights
  • Network topology structure

Changes are automatically detected and reloaded!

TROUBLESHOOTING
===============

Q: "No routes calculated yet" when clicking a router
A: Click "Run [PROTOCOL]" first to calculate routes

Q: Animation too fast/slow
A: This is protocol-specific behavior (RIP should be slowest)

Q: Want to see different protocols?
A: Edit main.py, change PROTOCOL_NAME, and restart

Q: Window closing with error?
A: This is normal - just close the terminal window

PROTOCOL COMPARISON
===================

Convergence Speed:  OSPF/IS-IS > BGP > RIP
Hop Limitation:     Only RIP (15 hops max)
Animation Speed:    OSPF/IS-IS (Fast) → BGP (Medium) → RIP (Slow)
Table Format:       Each protocol uses native format
Hierarchical:       Only IS-IS (L1/L2 levels)
Policy-Based:       Only BGP
